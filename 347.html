Begriffserklärungen

  • Containerisierung: Schreibgeschützte Vorlage mit dem Speicherbild eines Containers.
  • Image: Aktive Instanz eines Images.
  • Layer: Teil eines Images, der einen Befehl oder eine Datei enthält.
  • Repository: Gruppierung von gleichnamigen Images mit verschiedenen Tags.
  • Registry: Verwaltet Repositories, z.B. DockerHub.

Unterschied zwischen Virtualisierung und Cloud

  • Virtualisierung dreht sich um das Teilen eines einzelnen Computers in mehrere Teile, während Cloud Computing darum geht, Programme und Daten über das Internet auf Servern auszuführen und zu speichern.

Sie können verschiedene Versionen (Tags) eines Container-Images nutzen

Dies ist meistens die Version des Images, beliebt sind “latest” oder die Version “22.04”.

Elementare Docker-Befehle

  • Sie kennen die grundlegenden Befehle zum Erzeugen, Starten, Beenden und Löschen von Images und Containern.
  • Sie können die wichtigsten Optionen von docker run (--name, --rm, --network, --ip, -d, -it, -p, -v, -e) korrekt anwenden. Siehe Optionen

Docker Befehle

Jeder Befehl in der Shell beginnt mit "docker". Nach dem Doppelpunkt kommt immer die Version des Images.

Simple Docker-Befehle

Beschreibung Docker-Befehl
Image herunterladen docker pull ubuntu:latest
Images anzeigen docker images
Container starten (allg.) docker run
Spezifischen Container starten docker run -it[shell] --name [container_name] [image]
Alle gestarteten Container anzeigen docker ps
Shell verlassen vom Container exit
Container wieder starten nachdem er mit Exit gestoppt docker start -i [container_name]
Container stoppen docker stop [container_name]
Container löschen, jedoch bleibt das Image vorhanden docker rm [container_name]
Image löschen docker rmi [image_name:tag]
Zusammensetzung von Image Namen source/imagename:tag
Einen Befehl in einem Container direkt ausführen docker exec [Optionen(-it)] [container_name] [Befehl(/bin/bash)]

Docker-Optionen

Beschreibung Docker-Befehl
Sie geben Ihrem Container einen Namen. docker run --name [container_name] [image_name]
Container automatisch entfernen nach dem Beenden. docker run --rm [image_name]
Bestimmen, welchem Netzwerk der Container angehört. docker run --network [network_name] [image_name]
Festlegen einer festen IP-Adresse für den Container. docker run --ip 192.168.0.10 [image_name]
Starten des Containers im Hintergrund. docker run -d [image_name]
Starten einer interaktiven Sitzung im Container. docker run -it [image_name]
Weiterleitung von Ports zwischen Host und Container. docker run -p 8080:80 [image_name]
Hinzufügen von Volumes zum Container. docker run -v /host/directory:/container/directory [image_name]
Setzen von Umgebungsvariablen im Container während der Ausführung. docker run -e "VAR=value" [image_name]
